**On this page**, you will need to introduce your worklog. **How** did you complete it? How many weeks did you take to complete the program? **What** did you do in those weeks?

Typically, and as a standard, a worklog is carried out over about 3 months (throughout the internship period) with weekly contents as follows:

**Week 1:** [Getting familiar with the AWS Account, AWS CLI, and Amazon EC2 & EBS](1.1-week1/)

**Week 2:** [Access management with IAM, network design with Amazon VPC, and object storage with Amazon S3](1.2-week2/)

**Week 3:** [Automatically scaling resources with EC2 Auto Scaling and building Serverless applications with AWS Lambda](1.3-week3/)

**Week 4:** [Containerizing applications with Docker/Amazon ECS and building a CI/CD pipeline with AWS CodePipeline](1.4-week4/)

**Week 5:** [Security & governance with AWS CloudTrail and AWS Config, and system monitoring with Amazon CloudWatch](1.5-week5/)

**Week 6:** [Shared file storage with Amazon EFS and automated infrastructure administration with AWS Systems Manager](1.6-week6/)

**Week 7:** [Cost Optimization and building a backup & recovery solution with AWS Backup](1.7-week7/)

**Week 8:** [Data-stream analytics with Amazon Kinesis and Athena, and database migration with AWS DMS](1.8-week8/)

**Week 9:** [Advanced networking with AWS Transit Gateway and generative AI applications with Amazon Bedrock](1.9-week9/)

**Week 10:** [Infrastructure as Code with AWS CloudFormation, caching with Amazon ElastiCache, and building the group capstone project](1.10-week10/)

**Week 11:** [Finalizing the group project, advanced cost management, and real-world troubleshooting](1.11-week11/)

**Week 12:** [Final report defense, product handover, and wrapping up the internship](1.12-week12/)
